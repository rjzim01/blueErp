EXISTING DAMAGES MODULE IN blueErp:
-----------------------------------
The old blueErp project has a complete Damages module:

1. DATABASE STRUCTURE:
   - Table: `damages` (defined in app/Models/Damages.php)
   - Primary Key: id
   - Config stored in tb_module table (module_name = 'damages')

2. MODELS:
   - /home/rifat/office/blueErpFolder/blueErp/app/Models/Damages.php
     * Extends Sximo base model
     * Table: damages
     * Has querySelect, queryWhere, queryGroup methods

3. CONTROLLERS:
   - Web Controller: app/Http/Controllers/DamagesController.php
     * Module: 'damages'
     * Standard CRUD operations (index, create, edit, show, store, destroy)
     * Uses makeInfo() to load module config from tb_module
   - API Service Controller: app/Http/Controllers/Services/DamagesController.php
     * Same operations but returns JSON responses

4. ROUTES:
   - Web: Route::resource('damages','DamagesController')
   - API: Route::resource('services/damages','Services\DamagesController')

5. VIEWS (Blade templates):
   - /resources/views/damages/index.blade.php - Listing page
   - /resources/views/damages/form.blade.php - Create/Edit form
   - /resources/views/damages/view.blade.php - Detail view
   - /resources/views/damages/public/index.blade.php - Public listing
   - /resources/views/damages/public/form.blade.php - Public form
   - /resources/views/damages/public/view.blade.php - Public detail

6. MODULE CONFIGURATION:
   - Stored in tb_module database table
   - Fields defined in module_config JSON column
   - Uses makeInfo() method to load config dynamically
   - Grid configuration for listing
   - Form configuration for CRUD

   - Uses Filament's table builders and form builders

2. CREATE DATABASE MIGRATION:

4. ADD SIDEBAR MENU:
